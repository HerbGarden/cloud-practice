useradd 디폴트 값
	creat_email_spool = yes 메일 자동 생성
	skel = 계정 생성시 자동으로 파일을 생성해줄 위치

	-b 디폴트 값을 바꾸는 것
	-d 계정 생성시 바꾸는 것

group
	생성시 groupadd 유저 : 그룹 생성
	-g 1500을 할시 GID를 1500으로 생성함

	group에 맨 마지막 위치에는 소속된 계정이 나옴
	안나올 경우 passwd에서 확인 가능한 경우 임
	
	-g 주 그룹 (한개만 가능)
	-G 보조 그룹(여러개 혹은 없어도 됨)

	groups 유저 : 속해있는 그룹을 볼수 있음

usermod로 -G 옵션으로 그룹을 바꿀경우 기존 것을 덮어 쓰므로
	  -aG 옵션을 주면 추가가 됨

groupdel로 삭제 가능
	그룹에 계정이 속해있으면 삭제 불가능


계정 설정 파일
 - /etc/login.defs 파일에 다 있음

시스템 계정을 root로 포함 가능

파일 타입
	-: 정규파일(일반파일)
	d: 디렉토리파일
	l: symbolic 링크 파일(mswindow에서 바로가기 파일 개념)

디렉토리에 실행 권한이 없으면 r권한이 있어도 제대로 볼 수 없음
*디렉토리는 x권한이 없으면 rw권한이 둘다 제대로 할수 잆다*

원하는 사람에게만 권한을 부여하기 위해서는 소유주를 바꾸는게 가장 빠름
 - chown 소유자 file명
 - chgrp 그룹 바꾸기

한번에 바꾸기 chown 소유자.(:)그룹 file명

su -c(command) '명령어' 사용자 : 사용자 권한으로 명령어를 실행하는 것


프로세스 관리
ps
ps 명령을 아무런 옵션 없이 사용하면 출력되는 내용은 현재 터미널에서(여기서는 pts/1) 현재 사용자 권한
(여기서는 lee)으로 실행(메모리에 있거나 cpu 자원을 할당받은)중인 프로세스만 출력된다. 아무런 옵션 없이
사용하면 가장 단순한 형태로 출력해준다.

uptime
 load average : 최근 1분 : 최근 5분 : 최근 15분 부하량의 평균
	1을 초과하면 부화가 많은것

ps
 -f : id 까지 자세히 보여줌
 -U : 사용자를 명시하면 그 사용자의 프로세스를 보여줌
 -e : 모든 프로세스를 보여줌

pstree : ps을 tree형태로 보여줌

top : 프로세스를 실시간 모니터링

pgrep -fl : ps에서 원하는 것을 간단하게 찾아 보기 위해 씀
